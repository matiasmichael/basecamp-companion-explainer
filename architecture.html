🏗️ Architecture & Memory

How Works

The memory system, identity files, context windows, and session management that power your companion. Understanding this is key to working together effectively.

The Core Toolbox

Your companion isn't just a language model — it has real tools on a real machine.

💾

Persistent Workspace

Files, memory, and configs that survive across sessions. Nothing gets lost when the conversation ends.

💻

Linux Shell

Full terminal access — git, python, node, curl, anything. Real code running on a real machine.

🌐

Web Browser

Headless Chromium for browsing, screenshots, scraping, and interacting with web applications.

🔍

Web Search

Live internet search and page reading. Research, fact-check, and gather information in real-time.

⏰

Cron Scheduler

Automated tasks on schedule. Morning briefings, reminders, data collection — all on autopilot.

💬

Messaging

Send and receive messages through Telegram, Slack, and other channels. Stay connected with your team.

Two Layers of Memory

Understanding this is the key to knowing when your companion remembers things — and when it doesn't.

1

Chat Context

Current conversation. Has a token window — old messages get compressed. Details can get lost.

→
2

Memory Files

Markdown files on disk. Permanent. Preferences, specs, and facts live here forever.

→
3

Recalled on Demand

Before answering, the companion searches files. If it isn't written down, it might not be found.

💡 The Key Insight

If you tell your companion something important and it doesn't get written to a file — future sessions may not know it. That's why "write it down" matters.

Why File-Based Memory Is Powerful

Most AI assistants have one type of memory: the conversation window. Once it gets long enough, the oldest messages fall off. Details are gone forever.

Your companion's memory works differently. The file system is the memory — and it has no upper limit. Every important fact gets written to markdown on disk. Those files don't expire, don't get compressed, and don't fall off any edge.

The insight: this turns memory into a search problem, not a capacity problem. Search is a solved problem that gets more efficient every day.

🎯 In Theory

Infinite memory with perfect recall. In practice, we're getting closer every day as search and file management improve.

Where It Can Break Down

✏️

The Write Problem

Memory is only as good as what gets written. If the companion doesn't save it, future sessions won't know. Say "save that" to lock it in.

📅

The Staleness Problem

Files can become outdated. No automatic expiry on a markdown file — it looks just as valid whether written yesterday or six months ago.

🔎

The Search-Not-Read Problem

Search depends on asking the right question. If a fact is filed under an unexpected heading, the companion might miss it.

⚠️

The Confabulation Problem

The biggest risk: thinking it remembers something when it's reconstructing from compressed summaries. The fix: always check the file.

Identity Files

These files define who your companion is and how it behaves. Loaded into the system prompt at session start — they shape how it thinks, talks, and acts.

🧬

SOUL.md

The deepest layer. Personality, communication style, values, boundaries. This is where "sharp and direct, no filler" or "warm and supportive" comes from.

# SOUL.md
## Personality
Direct, efficient, no filler.
Flags uncertainty explicitly.
## Communication Style
Concise bullets for updates.
Detailed narratives for analysis.

Controls: Tone, personality, values, guardrails

🪪

IDENTITY.md

The basics: name, model, emoji, role. Simple constants — think of it as the companion's birth certificate.

# IDENTITY.md
Name:
Emoji:
Role: AI Companion
Model: Claude Sonnet

Controls: Name, username, model, display info

👤

USER.md

Everything about the people the companion serves — names, timezones, preferences, working styles, communication quirks.

# USER.md
## Preferences
Timezone: PT (Los Angeles)
Briefing: 6:30 AM, max 5 items
Style: Bullet points for updates

Updated by: Telling your companion about yourself

📜

MEMORY.md

The persistent knowledge base. Facts, decisions, project context, preferences — everything important that should survive across sessions.

# MEMORY.md
## Decisions
- Use TypeScript for all new code
- Deploy to Vercel, staging first
## Key Dates
- Launch: March 15

Updated by: Companion writes automatically + "save that"

📝

HEARTBEAT.md

The companion's session journal. What it worked on, what's pending, what was accomplished. Updated at the end of each session to maintain continuity across conversations.

How to Shape Your Companion

No config files to learn. The best way to shape your companion is by talking to it about how you want it to behave.

💬 Just Say It

"Be more concise in briefings. Bullet points only."

"Don't use emojis in serious reports."

"I prefer you flag uncertainties explicitly."

"My timezone changed — I'm in NYC now."

✅ What Happens

The companion updates the relevant file (SOUL.md, USER.md, etc.) based on your instruction. The change takes effect in the current session and persists into all future ones.

No restart needed. No config editing. Just a conversation.

The Context Drift Problem

Identity files are loaded at session start. As conversations get longer, identity instructions get proportionally smaller in the context window.

Session Start

~95% Identity

Identity files dominate the context. Companion behaves exactly as configured.

Mid-Session (~50k tokens)

~60% Identity

Conversation history starts competing. Still solid, but personality may soften slightly.

Long Session (~150k tokens)

~25% Identity

Identity is a small fraction. Companion may feel more "generic." Time for a fresh session.

💡 The Fix

If your companion starts feeling generic — it's drift. A fresh session resets it instantly. Or just say: "You're drifting, re-read your SOUL."

Session Management

How conversations work, reset, and maintain continuity.

🔄

Session Start

Identity files (SOUL, IDENTITY, USER) are loaded. HEARTBEAT.md is read for recent context. The companion is fully "itself" again.

💬

During Session

The conversation builds context. Important discoveries are written to memory files. The companion learns as you talk.

📝

Session End

HEARTBEAT.md is updated with what happened, what's pending, and what was decided. The next session picks up where this one left off.

🔑 Key Principle

The companion doesn't lose knowledge between sessions — it writes things down. The file system bridges the gap between sessions the same way a notebook bridges the gap between workdays.
