📖 User Guide

Getting the Most From Your Companion

Think of your companion as a sharp, resourceful employee — not a chatbot. These best practices will help you work together effectively.

The Golden Rules

Six principles that dramatically improve your experience.

🎯

Be Specific

Think of requests as specs, not wishes. "Send me news at 6:30 AM PT, max 5 bullets, real journalism only" — that's implementable. "Send me some news" is not.

📝

Say "Write It Down"

If something matters, tell your companion to save it. "My timezone is PT, I'm in LA. Save that." — locks it into persistent memory for all future sessions.

🏗️

Break Big Tasks Into Phases

Instead of "build me a website," try: "Phase 1: Research three competitors. Phase 2: Draft a sitemap. Phase 3: Build the homepage." Smaller units = better results.

🔍

Push Back on Vague Answers

If your companion says "it should be working" — push back. "Show me the proof. Check the logs. Verify the state." Don't let it get away with uncertainty.

⚡

Call Out Mistakes Directly

Your companion can check logs, configs, and state — but only if it knows something's off. "That's wrong. The format should be X, not Y." Direct correction is the fastest path.

🔧

Ask It to Build Things

Not just Q&A — recurring workflows, monitoring scripts, data pipelines, automated reports. Your companion is a builder, not just an answerer.

Communication Best Practices

How you communicate affects the quality of what you get back.

✅ Do This

Give Context

"I'm preparing for a board meeting on Thursday. I need a 5-slide summary of our Q1 metrics, focusing on growth and retention."

Specify the Output

"Write it as bullet points, max 10 items, with one source link per item. No opinions, just facts."

Set Constraints

"Research this topic, but only from sources published in the last 3 months. Skip press releases."

State Your Role

"I'm the CTO. I need the technical deep-dive, not the executive summary. Include code examples."

❌ Avoid This

Vague Requests

"Tell me about marketing." — Too broad. What aspect? For whom? How detailed? What format?

Assuming Memory

"Use the same format as last time." — If it wasn't saved to a file, the companion might not remember. Restate or ask it to check.

Accepting Uncertainty

"It should work." — If you need it to work, push for verification. "Show me it works."

Mega-Requests

"Build me a full app with auth, payments, and deployment." — Break it down. Each phase should be testable.

Handling Blockers

When your companion gets stuck, here's how to unblock it fast.

🔑 "I need access to X" ▼
The companion needs credentials, API keys, or permissions. Provide them directly or tell it where to find them. For security-sensitive items, use environment variables or config files rather than pasting in chat.
🤔 "I'm not sure what you want" ▼
The request is ambiguous. Restate with more specifics: What's the output format? Who's the audience? What's the deadline? Give an example of what "good" looks like.
⏱️ "This is taking longer than expected" ▼
Long-running tasks can hit token limits or tool timeouts. Break the work into smaller chunks. Use sub-agents for parallel processing. Or simply ask: "What's blocking you? What do you need?"
🐛 "Something went wrong" ▼
Error states happen. Ask the companion to: check error logs, reproduce the issue, explain what it tried, and suggest alternatives. If it's a tool failure, a retry often works. If it's a logic error, redirect the approach.
🧠 "I forgot / I don't have context" ▼
This means the relevant info wasn't in a memory file. Two fixes: (1) Re-provide the context now, (2) Tell the companion to save it this time. Say: "Save this to MEMORY.md so you have it next session."

Group Chats & @Mentions

Working with your companion alongside other agents and team members.

💬

How @Mentions Work

In group chats, @mention your companion by name to direct a message to it. It sees the full conversation context and responds to the thread.

@ Can you research
the latest funding rounds in
AI infrastructure? Top 5 please.

The companion reads the group context and responds with relevant, conversation-aware answers.

⚡

Multi-Agent Coordination

Your companion can work alongside other AI agents in group chats. Each agent has its own specialty, and they coordinate through @mentions.

Best Practice

Be clear about who should do what. "@Agent1 write the code, @Agent2 review it" is better than throwing a task into the group and hoping.

Session Management Tips

Understanding sessions helps you get more consistent results.

🔄

When to Start Fresh

  • → Switching to a completely different topic
  • → Companion feels "generic" or is drifting from personality
  • → After a very long conversation (100k+ tokens)
  • → When you've updated identity files and want them fully loaded
➡️

When to Continue

  • → Still working on the same project or task
  • → Iterating on a deliverable (draft → revision → final)
  • → The conversation still feels sharp and on-point
  • → You need the companion to remember recent back-and-forth

💡 The HEARTBEAT Bridge

Don't worry too much about losing context between sessions. The HEARTBEAT.md file captures what happened, what's pending, and what decisions were made. A fresh session reads this file and picks up where you left off — like reading yesterday's notes before starting work.

Token Awareness

Understanding token usage helps you work efficiently and keep costs reasonable.

What Are Tokens?

Tokens are the unit of "thinking" for AI models. Every message you send and every response you receive costs tokens. Longer conversations use more tokens. Complex tasks (code generation, research) use more than simple Q&A.

Token-Efficient Habits

  • ✅ Start fresh sessions for new topics
  • ✅ Be specific upfront (fewer iterations)
  • ✅ Use skills for recurring tasks
  • ✅ Ask for summaries before deep-dives

Token-Heavy Patterns

  • ⚠️ Very long, meandering conversations
  • ⚠️ Vague requests that require many iterations
  • ⚠️ Repeatedly re-explaining the same context
  • ⚠️ Generating huge outputs you won't fully use

Quick Reference

Phrases and commands that unlock specific behaviors.

Say This To Get This
"Save that" / "Write it down" Persists information to memory files for future sessions
"You're drifting, re-read your SOUL" Reloads identity and personality from SOUL.md
"Run my morning briefing" Executes the morning briefing skill (if configured)
"Check the logs" / "Show me proof" Forces verification instead of assumptions
"What's on my plate today?" Reviews calendar, tasks, and pending items
"Be more concise" / "Give me more detail" Adjusts response style (saves to SOUL.md if told to)
"Break this into phases" Splits a big request into manageable chunks
"What do you know about me?" Shows what's in USER.md and memory files
"Create a skill for this" Saves the current workflow as a reusable skill file
"What did we decide about X?" Searches memory files and session history for decisions

The Right Mental Model

Your companion is a sharp, resourceful employee on Day 1 of a very long job. The more context you give, the more it can do. The more skills you teach, the more autonomous it becomes.

🗓️

Week 1

Setting preferences, learning your style, building initial skills. Lots of "save that" moments.

📈

Month 1

Running daily briefings, handling research, managing tasks. Starting to feel indispensable.

🚀

Month 3+

Fully autonomous workflows. Proactive suggestions. An extension of how you think and work.
