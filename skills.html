📋 Skills System

Teach Once, Done Forever

Skills are reusable instruction sets — like SOPs, but ones that actually get followed. Teach your companion a workflow once, and it executes it perfectly every time.

What Are Skills?

Think of skills as recipes your companion can follow — step-by-step instructions stored as files, recalled and executed on demand.

📄

Skill Files

Each skill is a markdown file in the companion's workspace. It contains clear instructions: what to do, in what order, with what tools, and what the output should look like.

🎯

Not Suggestions — Procedures

Unlike "prompts" or "templates," skills are executed. The companion doesn't suggest what to do — it does it, following the skill file step by step.

# skill-morning-briefing.md

## Morning Briefing Skill

### Steps:
1. Check calendar for today's events
2. Search web for relevant industry news
3. Review pending tasks from MEMORY.md
4. Check GitHub for overnight PRs
5. Compile into briefing format

### Format:
- Max 5 news items
- Bullet points only
- Include source links
- Flag action items with ⚡

### Deliver:
- Send via Telegram at 6:30 AM
- Tone: concise, no fluff

Why Skills Matter

The difference between a useful companion and an indispensable one.

🔁

Consistency

The same process every time, no matter the session. No "I forgot how you like your reports" moments. The skill file is the source of truth.

📈

Compounding Value

Each skill you teach compounds. Your companion gets more capable not through updates, but through your investment in teaching it your workflows.

🚀

Delegation

Real delegation, not "generate a draft." You say "run my morning briefing" and get a complete, formatted, delivered result — no hand-holding needed.

Example Skills

Common skills that companions are taught. These are starting points — customize them to your exact needs.

☀️

Morning Briefing

Compile news, calendar events, pending tasks, and overnight notifications into a single digest. Deliver at your preferred time via your preferred channel.

📊

Weekly Summary

Review the past week's completed work, open loops, key metrics, and upcoming deadlines. Format as a concise executive summary.

📧

Email Triage

Scan inbox, categorize messages by urgency and topic, draft replies for routine items, flag anything that needs personal attention.

🗓️

Meeting Prep

Before meetings: gather context on attendees, review relevant documents, prepare talking points, set up note templates.

🔍

Deep Research

Multi-source web research on a topic. Cross-reference sources, extract key facts, compile into a structured report with citations.

📰

Competitive Intelligence

Monitor competitors' news, product updates, hiring patterns, and public announcements. Weekly digest with trend analysis.

📈

Market Scan

Track industry trends, funding rounds, acquisitions, and emerging technologies relevant to your space.

👤

Person Research

Before a meeting or outreach: compile background on a person — their work, interests, recent activity, mutual connections.

✍️

Blog Post Drafting

From outline to polished draft. Research the topic, structure arguments, write in your voice (learned from SOUL.md), include relevant examples.

🐦

Social Media

Create platform-appropriate content from your ideas. Thread format for X/Twitter, concise for LinkedIn, casual for Telegram.

📝

Document Editing

Review and improve documents: tighten prose, fix structure, ensure consistency, add missing context. Respects your style preferences.

📧

Email Drafting

Draft emails matching tone to recipient — formal for investors, casual for teammates, persuasive for sales. Always in your voice.

🔀

PR Review

Review pull requests: check for bugs, style issues, test coverage, security concerns. Post review comments directly on GitHub.

🐛

Bug Investigation

From error report to root cause. Search logs, trace code paths, identify the issue, suggest (or implement) the fix.

📦

Deployment Checklist

Pre-deployment: run tests, check staging, verify configs, create backup plan. Post-deployment: verify health, monitor errors.

📋

Sprint Planning

Gather open issues, prioritize by impact, estimate effort, draft sprint scope. Present as a clear plan ready for team review.

How to Teach New Skills

Teaching your companion is a conversation, not a configuration exercise.

1

Describe What You Want

Tell your companion what the workflow should do, step by step. Be as specific as you'd be training a smart new hire.

"Every Monday morning, I want you to:
1. Check our GitHub repos for open PRs
2. Summarize each PR in one line
3. Flag anything older than 3 days
4. Send me the list on Telegram"
2

The Companion Creates the Skill File

Your companion writes it as a structured markdown file, confirms the steps with you, and saves it to the workspace.

3

Test & Refine

Run the skill once, review the output. Too verbose? Not enough detail? Just tell it: "Add source links to each PR summary." The skill file gets updated.

4

Use Forever

The skill is now part of your companion's repertoire. Say "run the PR review" anytime — or schedule it to run automatically via cron.
